Content/Style Behaviour, HTML, CSS, JSServer vs client, Key concerns, History & future Cyclical design model

  • Content: Refers to the structure and elements of a webpage.
  • Style: Involves the presentation and layout using CSS.
  • Behaviour: Interaction and dynamic changes implemented using JavaScript.

HTML, CSS, JS:

  • HTML (Hypertext Markup Language): Defines the structure of web content.
  • CSS (Cascading Style Sheets): Controls the presentation and layout.
  • JS (JavaScript): Enables interactive and dynamic behavior.

Server vs Client

  • Server-Side: Processes occur on the webserver
  • Client-Side: Processes occur on the user's browser

DOM, Cross browser compatibility, Separation of concerns, Classes & IDs

DOM (Document Object Model)

  • Representation: A tree-like structure representing the document.
  • Manipulation: Using scripting languages to change document structure.

Cross Browser Compatibility:

  • Ensuring Consistency: Making sure the website functions similarly on different browsers.
  • Testing: Regular testing on various browsers during development.

Separation of Concerns:

  • Modularity: Dividing code into logical sections for maintainability.
  • HTML, CSS, JS Files: Each responsible for specific aspects.

Classes & IDs:

  • Classes: Groups of elements sharing common styles or behavior.
  • IDs: Unique identifiers for specific elements.

The cascade, Inheritance, Overrides, CSS resets, Background images

The Cascade:

  • Priority System: How styles are applied based on specificity and importance.
  • Inheritance: Styles inherited by child elements.

Overrides:

  • Specificity: Rules governing which style declarations take precedence.
  • !important: Forceful declaration.

CSS Resets:

  • Normalization: Ensuring consistent styling across different browsers.
  • Reset Stylesheet: Zeroing out default styles.

Images, Links, Lists, Forms

Images

  • Embedding Images
  • Attributes: src, alt, width, height

Links

  • Creating hyperlinks
  • Attributes: href, target

Forms

  • Collecting user input
  • Validation: Ensuring data integrity

Relative vs absolute paths, Domain name and hosting, Deployment methodologies

Relative vs Absolute Paths:

  • Relative Paths: Defined in relation to the current page.
  • Absolute Paths: Defined with the full URL or directory structure.

Domain Name and Hosting:

  • Domain Name: The web address used to access a site.
  • Hosting: Providing server space to store and serve website files.

Deployment Methodologies:

  • Manual Deployment: Uploading files via FTP or other methods.
  • Automated Deployment: Using tools like Git or CI/CD pipelines for efficient deployment.

Positioning, Text layout, CSS floats

Positioning:

  • CSS Position Property: Specifies the positioning method.
  • Position Values: static, relative, absolute, fixed, sticky.

Text Layout:

  • Typography: Styling and arranging text on a webpage.
  • Line Height, Letter Spacing, and Font Styling.

CSS Floats:

  • Float Property: Aligns elements to the left or right.
  • Used for creating text wraps and layouts.

Responsive design, Adaptive layout, CSS media queries, Layout, Frameworks

Responsive Design:

  • Fluid Grids and Flexible Images.
  • CSS Media Queries: Adjust styles based on device characteristics.

Adaptive Layout:

  • Design that adapts to different screen sizes.
  • Customizing layouts for specific devices.

CSS Media Queries:

  • Conditional styling based on viewport characteristics.
  • Commonly used for responsive web design.

Layout Frameworks:

  • Bootstrap, Foundation, and other front-end frameworks.
  • Pre-built components for rapid development.

What is behaviour Examples Variables Operators Default functions Output

What is Behavior?

  • Interactive and dynamic actions on a webpage.
  • Manipulating the Document Object Model (DOM).

Examples:

  • Event Handling: Responding to user actions.
  • Form Validation: Checking user input for correctness.

Variables:

  • Containers for storing data values.
  • Declared using var, let, or const.

Operators:

  • Arithmetic, Comparison, Logical, and Assignment operators.
  • Used for performing operations on variables.

DOM manipulation Inspector Functions If statements

DOM Manipulation:

  • Changing the structure, style, or content of a webpage.
  • Accessing and modifying DOM elements using JavaScript.

Inspector:

  • Browser tools for debugging and inspecting web elements.
  • Examining and modifying HTML and CSS in real-time.

Functions:

  • Reusable blocks of code with a specific purpose.
  • Declared using function keyword, can accept parameters.

If Statements:

  • Conditional statements for decision-making in code.
  • Executing different blocks of code based on a condition.

Loops Arrays Testing

Loops:

  • For, While, and Do-While loops for repetitive tasks.
  • Iterating over arrays or a sequence of actions.

Arrays:

  • Ordered lists used to store multiple values.
  • Indexed, allowing easy access and manipulation of data.

Testing:

  • Verification of code functionality and performance.
  • Unit testing, integration testing, and user acceptance testing.

Testing Image compression Redirects/http requests Caching Minification Content delivery

Testing:

  • Ensuring website functionality and performance meet requirements.
  • Various testing methods: unit testing, integration testing, and user acceptance testing.

Image Compression:

  • Reducing image file sizes without compromising quality.
  • Improving webpage load times and overall performance.

Redirects/HTTP Requests:

  • Managing server requests and optimizing page redirects.
  • Minimizing unnecessary HTTP requests for faster loading.

Caching:

  • Storing static files locally to reduce load times for returning visitors.
  • Utilizing browser caching and server-side caching strategies.

Minification:

  • Removing unnecessary characters from code to reduce file size.
  • Improving load times by optimizing CSS, JavaScript, and HTML.

Testing Analytics Meta tags Title tags Sitemaps

Testing:

  • Evaluating website performance in search engine results.
  • Using tools like Google Search Console for insights.

Analytics:

  • Collecting and analyzing data about website traffic and user behavior.
  • Utilizing tools like Google Analytics for comprehensive insights.

Meta Tags:

  • Providing metadata to describe the content of each webpage.
  • Includes title tags, meta descriptions, and meta keywords.

Group Project Overview

Below you will find the details of the group project for this module outlined clearly and concisely for ease of understanding


Project Brief

You are required to develop a website for a business of your choice. This assignment is a group-based project, and each group will consist of 2-3 participants.

Website Characteristics

  • Minimum Web Pages:
    • For groups of 2: Minimum 4 web pages
    • For groups of 3: Minimum 6 web pages
  • All Pages Styled by One External CSS File
  • Clearly Defined, Consistent Navigation System and Form
  • Form Entries Should Be Validated
  • Some HTML Content to Be Manipulated Using JavaScript
  • Bootstrap May Be Used
  • jQuery May Be Used

Website Requirements

  • Deployment of the Website
  • Validated and Optimized HTML and CSS Syntax
  • Implementation of Search Engine Optimization Practices

Note: Customize the content based on the specific requirements of your project.

Design Requirements

You have been assigned as web developer for and you need to state the client requirements. In your design (wireframes & mood board) show how you incorporated the client requirements. Demonstrate the following:

  • Header with website title, logo and tagline.
  • Consistent navigation and design across all pages of the website.
  • At least 1 (or more) use of a background image.
  • Gallery of 4 or more images. When the user clicks an image, JavaScript to be used to show some text relating to that image.
  • Form with at least 4 fields of different types such as text, numeric, etc. Form to include a checkbox / radio / dropdown field. Include JavaScript validation.
  • All pages must have an example of JavaScript to demonstrate manipulation of the DOM.
  • A JavaScript minigame for visitors to play that includes an element of randomisation.
  • All pages are responsive.
  • There is only 1 external CSS file for all pages.
  • Use of folders and file paths.
  • Show the use of 2-3 Google fonts throughout site.
